## 1. Tổng quan codebase

| Hạng mục | Nội dung |
|----------|----------|
| **Stack** | Node.js + Express 4, MongoDB (Mongoose 9), JWT (`jsonwebtoken`), bcrypt, express-validator, nodemailer, slugify; `app.js` khai báo view engine **Pug** |
| **API prefix** | `/api/v1/...` cho `users`, `auth`, `roles`, `products`, `categories`, `carts` — xem `app.js` |
| **Phạm vi repo** | Backend API; không có frontend trong repo |
| **Entry** | `bin/www` → `require('../app')`, listen `process.env.PORT \|\| 3000` |
| **Cấu trúc chính** | `app.js`, `bin/www`, `routes/`, `controllers/users.js`, `schemas/`, `utils/` |

**Phân loại thư mục / vai trò**

- **Core:** `app.js`, `bin/www`
- **Business logic:** chủ yếu trong **route handlers**; `controllers/users.js` chỉ bọc thao tác user
- **Infra:** `mongoose.connect('mongodb://localhost:27017/NNPTUD-C6')` trong `app.js` (hardcode, không đọc env)
- **Shared:** `utils/authHandler.js`, `utils/validatorHandler.js`
- **Dead / suspicious:** `utils/data.js` (JSON catalog lớn, không thấy `require` từ file `.js` khác); `schemas/payments.js` gần như chỉ comment; `utils/IncrementalIdHandler.js`, `utils/GenToken.js` không được route require; `schemas/reservations.js` có model nhưng **không** mount trong `app.js`

**Ghi chú repo:** tại thời điểm phân tích, root **không có** thư mục `views/` và `public/` trong khi `app.js` vẫn `set('views', ...)` và `express.static('public')` — rủi ro khi `res.render('error')` hoặc static.

## 2. Kiến trúc và luồng chạy

### Khởi động

1. `bin/www`: `http.createServer(app)`, port `process.env.PORT \|\| 3000`
2. `app.js`: middleware (morgan, json, urlencoded, cookie-parser, static), connect MongoDB, mount routers, 404 → error handler (`res.render('error')`)

### Auth

- Cookie `token_login_tungNT` hoặc header `Authorization: Bearer <token>`
- `utils/authHandler.checkLogin` → `jwt.verify(..., "secret")` → `userController.FindById` (populate `role`) → `req.user`
- `CheckPermission("ADMIN")`: so `req.user.role.name` với chuỗi truyền vào (`routes/users.js` GET `/`)

### Dữ liệu (quan hệ chính)

- User ↔ Role (ObjectId ref `role`)
- Product ↔ Category; Inventory 1-1 Product
- Cart 1-1 User, line items `{ product, quantity }`
- **Payment / reservation:** `schemas/payments.js` chưa export model; `schemas/reservations.js` có model `reservation` nhưng **không** có `app.use` tương ứng — luồng nghiệp vụ chưa gắn API

### Thứ tự route (products/categories)

- Định nghĩa `GET /slug/:slug` **trước** `GET /:id` — tránh nhầm `slug` thành id (ổn nếu client dùng đúng pattern).

## 3. Logic nghiệp vụ chính

| Module | Vai trò |
|--------|---------|
| `routes/auth.js` | Đăng ký (transaction: user + cart), login (JWT cookie), `/me`, đổi MK, logout, forgot/reset password |
| `routes/users.js` + `controllers/users.js` | CRUD user — pattern không đồng nhất (có chỗ controller, có chỗ gọi model trực tiếp nhưng thiếu import) |
| `routes/products.js` | List/filter (filter phía app sau `find`), CRUD; tạo product + inventory trong transaction |
| `routes/categories.js` | CRUD category |
| `routes/carts.js` | GET giỏ, add/remove; đọc `inventory.stock` — **code không trừ kho** khi add/remove (chỉ kiểm tra) |
| `routes/roles.js` | CRUD role |

**So sánh hai luồng tạo user**

- **Đăng ký** (`auth.js`): `CreateAnUser(username, password, email, "<roleObjectId>")` — **khớp** 4 tham số đầu của `controllers/users.js` (`username, password, email, role`).
- **POST `/api/v1/users`** (`users.js`): gọi `CreateAnUser` với thứ tự **không khớp** chữ ký controller (`role` vs `fullName`/`avatarUrl` bị lệch vị trí) — xem mục 5.

## 4. Style / convention

- Lẫn `var` / `let`; comment Việt/Anh; message lỗi tiếng Việt không dấu ở nhiều chỗ auth
- Chỉ một file `controllers/`; product/category/role xử lý trực tiếp trong `routes/`
- Response không thống nhất: JSON object vs string; validation trong `handleResultValidator` trả HTTP **404** (không điển hình cho lỗi client)
- Import thừa: ví dụ `routes/carts.js` — `const { route } = require('./products')` không dùng (chỉ side-effect load router products)

## 5. Điểm bất thường / lỗi nghi ngờ

### Runtime / logic (ưu tiên)

1. **`routes/users.js`:** dùng `userModel` nhưng **không** `require('../schemas/users')` → `ReferenceError` khi chạy handler GET `/:id`, POST `/`, PUT `/:id`, DELETE `/:id`. (File vẫn `require` được vì lỗi chỉ lúc **thực thi** handler.)
2. **`routes/products.js`**, **`routes/categories.js`:** query dạng `{ _id: req.params.id && isDeleted }` — `isDeleted` **không** phải field trong object mà là **tên biến không tồn tại** → **ReferenceError** khi gọi GET `/:id`. Khả năng cao ý định: `{ _id: req.params.id, isDeleted: false }`.
3. **`routes/users.js` POST:** thứ tự đối số `CreateAnUser` **không khớp** `controllers/users.js` (controller: `username, password, email, role, fullname, avatar, status, logincount`; route truyền `fullName`, `avatarUrl`, `role` lệch vị trí so với chữ ký).
4. **`routes/auth.js` đổi mật khẩu:** gán `user.password` nhưng **không** `await user.save()` → mật khẩu có thể không lưu; hash trong `schemas/users.js` `pre('save')` chỉ chạy khi `save()`.
5. **`utils/authHandler.js`:** nhánh Bearer: sau `res.status(403).send(...)` khi thiếu token/định dạng — **thiếu `return`** → có thể chạy tiếp `token.split(...)` hoặc double response.
6. **`routes/auth.js` `POST /resetpassword/:token`:** nhánh thành công có thể thiếu `res` rõ ràng (nên xác nhận bằng HTTP thật).
7. **`utils/validatorHandler.js` — `RegisterValidator`:** `body('password').notEmpty().withMessage()` — `withMessage()` không có chuỗi → message validation có thể rỗng/undefined.
8. **`routes/carts.js`:** import không dùng từ `./products` (xem trên).

### Kiến trúc / hợp đồng

- `userUpdateValidator`: `password` không `.optional()` — PUT user có thể bị bắt buộc password mạnh dù chỉ sửa field khác
- Error handler toàn app: `res.render('error')` trong khi API chủ yếu JSON — kết hợp thiếu `views/` có thể làm lỗi phụ khi 500

### Edge case (cần xác nhận nghiệp vụ)

- **Giỏ hàng null:** nếu user chưa có document cart, `cart.products` có thể throw — luồng đăng ký trong `auth.js` tạo cart nên phụ thuộc dữ liệu DB.

### Bảo mật / cấu hình (ghi nhận)

- JWT secret `"secret"` hardcode; MongoDB URL hardcode trong `app.js`
- `utils/senMailHandler.js`: `auth.user` / `pass` rỗng — gửi mail có thể thất bại

### File lệch phong cách / orphan

- `schemas/payments.js`: chỉ comment, không export schema
- `schemas/reservations.js`: biến nội bộ tên `cartSchema` dễ nhầm; model không gắn route

## 9. Đề xuất hướng sửa an toàn (chưa thực hiện)

1. `routes/users.js`: thêm `require('../schemas/users')` **hoặc** chỉ dùng `userController` thống nhất; sau đó sửa thứ tự tham số `CreateAnUser` cho khớp `controllers/users.js`.

3. Đổi mật khẩu: `await user.save()` sau khi gán password (và xử lý lỗi).
4. `authHandler`: `return` sau mọi `res.status(403).send(...)`.
5. Chuẩn hóa mã HTTP validation (404 → 400): **breaking** — cần đồng bộ client.
